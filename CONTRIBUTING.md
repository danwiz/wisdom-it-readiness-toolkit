# Contributing

Contributions should be focused, documented, and safe by default.

1. Open an issue describing the problem or proposed improvement.
2. Keep assessment changes traceable to a clear rationale.
3. Add or update tests for scoring and reporting behavior.
4. Do not include real client data, credentials, or private records.
5. Run `python -m unittest discover -s tests -v` before submitting changes.

A contribution is complete when documentation, tests, security considerations, and backward-compatibility impacts have been addressed.
